## Preparation Philosophy

Most backend interviews **do not require competitive programming depth**.

Focus on:

- Recognizing common **problem patterns**
- Solving **easy → medium problems**
- Explaining the approach clearly
- Writing **clean, readable code**

Avoid spending time on:

- Advanced Dynamic Programming
- Segment Trees
- Heavy Graph Algorithms
- Competitive programming tricks

Goal: **Maximum interview coverage with minimal problems.**

## Learning Strategy

Main problem: **forgetting concepts after practice**.

Solution:

Learn → Practice → Revisit → Repeat

Instead of solving hundreds of questions, repeatedly practice a **small curated set**.
